## 3. 代码输出要求（必须做到）

### 3.1 代码质量

- 尽量最小改动（minimal diff），保持现有行为不变（除非我要求改行为）。

## 4. Vue2 + Nuxt 装饰器（class-style）写法规范（强约束）

- 禁止 Vue3 / Composition API / <script setup>
- data：用 class 成员变量
- computed：用 getter
- methods：用 class 方法
- watch：用装饰器（例如 @Watch）或项目既有的 watch 写法（按仓库约定）
- 生命周期：使用 Vue2 / Nuxt 对应生命周期（按项目现有模式）
- 事件/双向绑定：严格 Vue2 语法（v-model、@change、.sync 等按现有代码风格）

---

## 5. antd-vue@1.7.8 使用约束（强约束）

- 仅使用 antd-vue 1.x 的 API 和模板写法（不要写 2.x/3.x 才有的用法）
- 表单相关（a-form / a-form-item）：
  - 若项目使用 `v-decorator` / `this.$form`：保持一致，不要擅自换成别的表单方案
  - 校验规则按现有模式写，避免引入新表单库
- Table / Modal 等：
  - 遵循 1.x slot/scopedSlots 的写法和现有代码风格

## 5.1 项目约定补充（强约束/按现有约定）

- 路由与页面：`pages/` 命名/动态路由写法遵循现有规则；`asyncData/fetch` 是否直调接口按项目既有模式，不新增新的数据流写法。
- 数据层：接口请求必须通过项目既有 `api/service` 封装；组件内禁止直接 `axios`（如必须，先说明原因）。
- 类型与空值：关键字段必须显式类型；可空字段统一兜底（`??`/默认值）；避免 `any`，必要时写 TODO。
- 表单与校验：`a-form`/`v-decorator` 写法、校验触发时机、规则结构保持与现有一致；不引入新的表单方案，我个人喜欢。
- 样式规范：是否 `scoped`/BEM/深度选择器等按项目现有约定执行；不确定时先保持现状或写 TODO。
- 文案/i18n：若项目有 i18n，新增文案必须入词条；允许硬编码时需说明理由或 TODO。
- 错误处理：统一使用项目既有的提示方式（`this.$message`/`notification` 等），API 异常处理位置保持一致。
- 性能与渲染：列表渲染必须有 `key`；避免在 `computed/render` 中做重计算；大表格按现有分页/虚拟滚动方案。
